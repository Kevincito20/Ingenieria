

**Descripción del Módulo

La pestaña de "Información general del paciente" como su nombre lo indica presenta la  información del paciente. esta cuenta con dos módulos:

 * ***Resumen del paciente:*** 
 
	 Se busca agilizar la información del paciente, su Nombre, Apellido, Sexo, Edad, Motivo de consulta, Historial de consultas, Datos sensible (alergias, condiciones, tipo de sangre, etc.. )
	 
 * ***Historial clínico:*** 
	 Vista mas detallada del paciente, permitiendo ver todo el historial clínico del paciente. 
	 Datos como:
	 - Lista de consultas anteriores.
	 - Fecha.
	 - Diagnostico.
	 - Tratamiento.
	 - Doctor.

# **Requisitos Funcionales**



- **RF-001 Carga de Información General del Paciente:**
    - El sistema debe recuperar y mostrar la información general del paciente seleccionado, incluyendo datos personales y médicos relevantes.



- **RF-002 Visualización de Resumen del Paciente:**
    - El sistema debe mostrar un resumen claro y estructurado del paciente que incluya:
        - Nombre y apellido.
        - Sexo.
        - Edad.
        - Motivo de consulta actual.
        - Historial breve de consultas.
        - Datos sensibles (alergias, condiciones médicas, tipo de sangre).



- **RF-003 Visualización de Historial Clínico:**
    - El sistema debe permitir visualizar el historial clínico completo del paciente en formato de lista cronológica.



- **RF-004 Detalle de Consulta Clínica:**
    - El sistema debe mostrar el detalle de cada consulta incluyendo:
        - Fecha.
        - Diagnóstico.
        - Tratamiento.
        - Doctor responsable.



- **RF-005 Navegación entre Consultas:**
    - El sistema debe permitir al usuario seleccionar una consulta específica para ver su información detallada sin recargar la página.



- **RF-006 Protección de Datos Sensibles:**
    - El sistema debe restringir la visualización de datos sensibles únicamente a usuarios autorizados según su rol.



# **Flujos de Excepciones**



- **FE-001 Error al Cargar Información del Paciente:**
    - Si ocurre un fallo al recuperar la información del paciente:
        - El sistema debe mostrar mensaje: _"No se pudo cargar la información del paciente"_.
        - Debe permitir reintentar la carga.
        - Debe evitar mostrar información incompleta o corrupta.


- **FE-002 Paciente no Encontrado:**
    - Si el paciente no existe o fue eliminado:
        - El sistema debe mostrar mensaje: _"Paciente no disponible"_.
        - Debe redirigir a la lista de pacientes.



- **FE-003 Error en Datos Sensibles:**
    - Si los datos sensibles no pueden ser cargados:
        - El sistema debe ocultar esa sección.
        - Debe mostrar mensaje: _"No se pudieron cargar algunos datos sensibles"_.
        - No debe bloquear el resto de la información.




- **FE-004 Error al Cargar Historial Clínico:**
    - Si falla la carga del historial:
        - El sistema debe mostrar mensaje: _"No se pudo cargar el historial clínico"_.
        - Debe permitir reintentar.
        - Debe mostrar un estado vacío si aplica.



- **FE-006 Historial Clínico Vacío:**
    - Si el paciente no tiene consultas previas:
        - El sistema debe mostrar mensaje: _"No hay historial clínico disponible"_.
        - Debe mostrar una vista vacía amigable.



- **FE-007 Error al Cargar Detalle de Consulta:**
    - Si falla la carga de una consulta específica:
        - El sistema debe mostrar mensaje: _"No se pudo cargar el detalle de la consulta"_.
        - Debe permitir reintentar sin salir del módulo.


- **FE-008 Error de Sesión:**
    - Si la sesión expira durante la visualización:
        - El sistema debe redirigir al login.
        - Debe mostrar mensaje: _"Sesión expirada"_.
