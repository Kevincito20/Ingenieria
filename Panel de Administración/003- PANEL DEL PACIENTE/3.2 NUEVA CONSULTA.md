
**Descripción del módulo "Nueva consulta":

Diseñada para que el doctor pueda registrar una atención médica de forma rápida, pensado para priorizar el ingreso de información clínica durante la consulta.

Esta cuenta con dos módulos:

- **Formulario de consulta:
	- Permite registrar la información principal de la atención medica actual, diseñada para ser rápida y fácil de completar.
		- DATOS:
			1. Motivo de consulta.
			2. Síntomas.
			3. Diagnostico.
			4. Tratamiento.
			5. Notas adicionales.

- **Acciones de registro: 

	Se encarga de gestionar las  acciones relacionadas con la consulta registrada, permitiendo guardar la información y continuar con el flujo clínico. 
	
	- Funciones:
		1. guardar consulta.
		2. guardar y generar receta electrónica.
		3. validación de campos obligatorios
		4. confirmación de registro exitoso.


---
# **Requisitos Funcionales**



- **RF-001 Registro de Nueva Consulta:**
    - El sistema debe permitir al doctor registrar una nueva consulta médica asociada a un paciente activo.



- **RF-002 Validación de Campos Obligatorios:**
    - El sistema debe validar que los campos obligatorios (motivo de consulta, diagnóstico y tratamiento) estén completos antes de permitir el registro.



- **RF-003 Guardado de Consulta:**
    - El sistema debe permitir guardar la consulta médica en la base de datos de forma segura.



- **RF-004 Guardar y Generar Receta Electrónica:**
    - El sistema debe permitir guardar la consulta y generar una receta electrónica asociada en un mismo flujo.


- **RF-005 Confirmación de Registro:**
    - El sistema debe mostrar una confirmación visual clara indicando que la consulta fue registrada exitosamente.



- **RF-006 Edición Antes de Guardar:**
    - El sistema debe permitir modificar cualquier campo antes de confirmar el registro de la consulta.


---

# **Flujos de Excepciones**



- **FE-001 Campos Obligatorios Vacíos:**
    - Si el usuario intenta guardar sin completar campos obligatorios:
        - El sistema debe resaltar los campos faltantes.
        - Debe mostrar mensaje: _"Complete los campos obligatorios"_.
        - No debe enviar la información al backend.



- **FE-002 Error al Guardar la Consulta:**
    - Si ocurre un fallo al guardar la consulta:
        - El sistema debe mostrar mensaje: _"No se pudo guardar la consulta"_.
        - Debe permitir reintentar.
        - Debe conservar los datos ingresados.




- **FE-003 Pérdida de Conexión:**
    - Si se pierde la conexión durante el registro:
        - El sistema debe notificar: _"Sin conexión. Intente nuevamente"_.
        - Debe conservar los datos en el formulario.
        - Debe permitir reintento cuando se restablezca la conexión.



- ==**FE-004 Sesión Expirada:**== 
    - Si la sesión expira durante el registro:
        - El sistema debe redirigir al login.
        - Debe mostrar mensaje: _"Sesión expirada"_.
        - Debe intentar preservar los datos si es posible.



- **FE-005 Duplicidad de Registro:**
    - Si se detecta un intento de registrar la misma consulta múltiples veces:
        - El sistema debe prevenir duplicados.
        - Debe mostrar mensaje: _"Esta consulta ya fue registrada"_.



- **FE-008 Error del Servidor:**
    - Si ocurre un error interno del sistema:
        - El sistema debe mostrar mensaje: _"Error interno, intente más tarde"_.
        - Debe registrar el evento para monitoreo.
