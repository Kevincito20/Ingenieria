## 1. Cabecera (Header) & Notificaciones

_Cumple con [[1. Requisitos funcionales#RFI-004 Notificaciones del Ciclo de Vida de Recetas|RFI-004 Notificaciones del Ciclo de Vida de Recetas]], [[1. Requisitos funcionales#RFI-006 Identidad y Validación KYC|RFI-006 Identidad y Validación KYC]]_
- **Layout:** Fila superior (`flex-direction: row`, `justify-content: space-between`).
    
- **Izquierda:** Saludo dinámico `Hola, {user.firstName}`.
    
- **Derecha (Acciones):**
    
    - **Campana de Notificaciones:** Con un _badge_ numérico o punto rojo si hay alertas no leídas (ej. vencimiento de receta, reabastecimiento).
        
    - **Avatar del Usuario:** Fotografía o iniciales.
        
    - **Estado KYC:** Si el usuario no está verificado, se superpone un icono de alerta crítica (⚠️) en la esquina inferior derecha del avatar. Al hacer tap, un _tooltip_ o modal debe indicar que los flujos de dispensación restringida están bloqueados.


## 2. Selector Global de Máquina Objetivo (Sticky/Fijo)

_Cumple con [[1. Requisitos funcionales#RFI-008 Selector Global de Máquina Objetivo|RFI-008 Selector Global de Máquina Objetivo]]_

Este es un componente crítico que condiciona el resto de la app. Te sugiero que sea un elemento persistente (tipo _banner_ o barra) justo debajo del Header.

- **UI:** Un botón ancho o selector desplegable que diga: `📍 Dispensador actual: [Seleccionar Máquina]`.
    
- **Comportamiento:** Al presionarlo, abre un _Bottom Sheet Modal_ que obliga al usuario a elegir una máquina física. Una vez seleccionada, el ID de esta máquina se guarda en el estado global para inyectarlo en las llamadas a la base de datos (por ejemplo, si estás gestionando el estado con Supabase, esto filtra todas las consultas de inventario).



## 3. Acceso Directo: Última Compra

_Cumple con  [[1. Requisitos funcionales#RFI-003 Acceso Directo a Última Compra|RFI-003 Acceso Directo a Última Compra]]_

- **Layout:** Una tarjeta horizontal destacada debajo del buscador.
    
- **Contenido:** Nombre del último producto comprado y fecha.
    
- **Acción:** Botón primario **"Repetir Pedido"**.
    
- **Lógica Condicional:** Este botón evalúa el stock del producto en la máquina global seleccionada. Si hay inventario, el botón está habilitado; si no, aparece deshabilitado (gris) con un texto auxiliar: _"Sin stock en esta máquina"_.



## 4. Módulo de Recetas Vigentes

_Cumple con [[1. Requisitos funcionales#RFI-005 Indicador Global de Recetas Vigentes|RFI-005 Indicador Global de Recetas Vigentes]] [[1. Requisitos funcionales#RFI-007 Módulo de Máquinas y Recetas Frecuentes|RFI-007 Módulo de Máquinas y Recetas Frecuentes]]_

- **Título:** "Tus Recetas Activas".
    
- **Indicador Visual (RFI-005):** Si el vector de recetas tiene elementos válidos para uso inmediato, colocar un punto verde parpadeante o un _badge_ que diga "Listas para retiro" al lado del título.
    
- **Layout:** Cuadrícula de 2 columnas (`flexWrap: 'wrap'`).
    
- **Tarjetas (Cards):**
    
    - Fondo blanco, bordes redondeados, sombra ligera.
        
    - **Título:** Medicamento Genérico.
        
    - **Subtítulo:** Institución (ej. _Hospital Santo Tomás_).
        
    - **Footer:** Fecha de expiración (en rojo si está próxima a vencer).
